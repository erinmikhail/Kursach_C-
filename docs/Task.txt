Реализация Storage-модулей
Задача 1: Реализация Пул строк (StringPool)

Исполнитель: [Дарья Полетаева]
Файлы для работы: include/storage/StringPool.hpp (уже написан и подписана каждая функция) и src/storage/StringPool.cpp (нужно создать).
Суть задачи

В нашей СУБД мы не храним текст в основной таблице, чтобы все записи были одного размера (8 байт). Весь текст лежит в отдельном словаре string_pool.bin. Твоя задача — реализовать класс, который переводит строку в ID (число) и обратно.
Что нужно написать в StringPool.cpp:

    Конструктор StringPool(filepath):
        Просто сохраняешь путь к файлу в переменную filepath_.

    Метод load():
        Вызывается при старте СУБД. Открывает файл string_pool.bin в бинарном режиме на чтение.
        Нужно прочитать файл до конца.
        Подсказка по формату на диске: Удобнее всего хранить строки так: [Длина строки (4 байта)] [Сами символы].
        Считываешь длину, затем считываешь символы в std::string. Добавляешь строку в id_to_string_ (вектор) и в string_to_id_ (словарь). Индекс в векторе будет являться её ID. Увеличиваешь next_id_.

    Метод get_or_add_string(str):
        Сначала ищешь str в словаре string_to_id_ (используй .find()). Если нашел — просто возвращаешь этот ID.
        Если строки нет:
            Присваиваешь ей новый ID (next_id_).
            Добавляешь её в кэш (string_to_id_ и id_to_string_).
            Открываешь файл string_pool.bin в режиме дозаписи (std::ios::app | std::ios::binary).
            Записываешь на диск длину строки (4 байта) и саму строку.
            Увеличиваешь next_id_ на 1.
            Возвращаешь новый ID.

    Метод get_string(id):

        Просто возвращаешь строку из вектора id_to_string_[id].

    Не забываем написать документация в docs папку, чтобы потом подготовить защиту курсовой работы. 




Реализация Менеджера записей (RecordManager)

Исполнитель: [Дарья Ротова]
Файлы для работы: include/storage/RecordManager.hpp (уже написан) и src/storage/RecordManager.cpp (нужно создать).
Суть задачи

У нас есть Pager, который умеет читать и писать куски данных строго по 4096 байт (страницы). Твоя задача — научить программу «нарезать» эти 4096 байт на строчки нашей таблицы. Одна структура Record весит ровно 8 байт (два числа по 4 байта). Следовательно, в одной странице помещается 4096 / 8 = 512 записей.
Что нужно написать в RecordManager.cpp:

    Конструктор RecordManager(pager):

        Сохраняешь ссылку на pager_.

    Арифметика (главное правило для остальных методов):

        Чтобы найти запись с индексом N (например, 1000-ю добавленную запись):

        Номер страницы = N / 512 (1000 / 512 = страница №1).

        Смещение в байтах = (N % 512) * 8 ((1000 % 512) * 8 = 3904-й байт от начала страницы).

    Метод get_record(record_index):

        Вычисляешь Номер страницы и Смещение.
        Запрашиваешь страницу у пейджера: std::vector<char> page = pager_.read_page(Номер страницы);

        Подсказка:
        Как вытащить 8 байт? Используй std::memcpy.

        Record rec;
        std::memcpy(&rec, &page[Смещение], sizeof(Record));
        return rec;

    Метод write_record(record_index, record):

        Вычисляешь Номер страницы и Смещение.
        Важно: Сначала читаешь всю страницу у пейджера (read_page), потому что нельзя записать 8 байт прямо на диск, можно перезаписать только страницу в 4096 байт целиком!

        Подсказка:
        Копируешь новые 8 байт в нужное место скачанной страницы:

        std::memcpy(&page[Смещение], &record, sizeof(Record));

        Отправляешь обновленную страницу обратно в пейджер: pager_.write_page(...).

    Метод append_record(record):

        Узнаешь текущее количество записей (это будет индекс для новой записи).
        Вычисляешь, на какой странице она должна лежать. Если эта страница еще не существует (выходит за пределы pager_.get_num_pages()), вызываешь pager_.allocate_page().
        Вызываешь свой же метод write_record(...) для сохранения.
        Увеличиваешь счетчик общего количества записей.
        Возвращаешь индекс вставленной записи.


    Не забываем написать документация в docs папку, чтобы потом подготовить защиту курсовой работы.
