Задание для Ильи: 

Мы закончили слой Storage, и теперь база умеет писать на диск. Твоя новая задача — сделать "лицо" нашей СУБД. Нам нужен модуль API, который будет общаться с пользователем через консоль.

    Что нужно сделать:
    Создать два класса в слое API: REPL (Read-Eval-Print Loop) и TableFormatter.

    1. Класс REPL (Главный цикл приложения):

    Должен выводить красивое приглашение в консоль, например: dbms > .

    Считывать строку ввода от пользователя.

    Если строка начинается с точки (например, .exit или .help), это системная мета-команда. Ее нужно обработать прямо здесь (завершить программу или вывести справку).

    Если строка не начинается с точки — мы считаем, что это SQL-запрос. Пока у нас нет парсера, просто выводи сообщение: "SQL запрос принят к обработке: [текст]".

    2. Класс TableFormatter (Отрисовщик таблиц):

    Для команды SELECT нам понадобится выводить данные красиво, а не просто слипшимся текстом.

    Нужно написать утилиту, которая принимает заголовки колонок и двумерный массив строк, а затем рисует псевдографическую таблицу.

    Она должна сама высчитывать ширину колонок в зависимости от самого длинного слова в ней, чтобы рамки не ехали.

    Пример того, что должен уметь выводить твой форматер:

    +----+---------+
    | id | name    |
    +----+---------+
    | 1  | Apple   |
    | 42 | Banana  |
    +----+---------+
    Файлы создавай в src/api/ и include/api/. Заголовочный файл с документацией (для нашей защиты) я тебе уже подготовил, используй его как основу!

Задание для Даги полетаевой: 
    Даша, привет! Мы начинаем делать парсер для нашей СУБД. Твоя задача — написать Lexer (Токенизатор).
    Наш движок не умеет читать сплошной текст. Твоя программа должна получить строку (например, INSERT 42 'Apple';) и порубить её на массив токенов (кубиков смысла), отбросив лишние пробелы.

    Что нужно сделать:

    Взять заголовочный файл Lexer.hpp, который я прикрепил. Там описана структура Token и типы токенов (KEYWORD, NUMBER, STRING и т.д.).

    Написать реализацию класса Lexer в Lexer.cpp. Главный метод — tokenize().

    Логика метода: идешь циклом while по строке.

    Видишь цифру? Читаешь всё число целиком -> создаешь токен NUMBER.

    Видишь одинарную кавычку '? Читаешь текст до закрывающей кавычки -> создаешь токен STRING.

    Видишь букву? Читаешь слово. Если это insert или select (в любом регистре) -> это KEYWORD. Остальное -> IDENTIFIER.

    Видишь пробел -> пропускаешь.
