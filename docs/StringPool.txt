StringPool (Пул строк)

Назначение:
StringPool управляет строками переменной длины, вынося их за пределы основных файлов таблиц базы данных. Он транслирует текстовые данные в уникальные числовые идентификаторы (ID) и обратно. Это позволяет модулю RecordManager оперировать записями строго фиксированного размера, что критически важно для производительности и реализации индексов (B-деревьев).

Основные структуры:

  - string\_to\_id\_ (std::unordered\_map\) - хэш-таблица для мгновенного поиска ID по тексту. Использует легковесные "представления" строк (string\_view) для исключения дублирования данных в оперативной памяти.
  - id\_to\_string\_ (std::deque[std::string]) - двусвязная очередь, хранящая сами строки. Индекс элемента в очереди является его ID. Использование deque (вместо vector) гарантирует, что адреса строк в памяти не изменятся при добавлении новых элементов, что необходимо для безопасной работы string\_view.
  - next\_id\_ - счетчик (uint32\_t), хранящий ID, который будет выдан следующей новой строке.
  - unsaved\_start\_index\_ - индекс, указывающий на начало блока новых строк, которые еще не были сброшены на жесткий диск.

Логика работы:

1.  Инициализация и Загрузка (load)
    При старте СУБД вызывается метод load(), который открывает бинарный файл словаря на чтение. Данные хранятся в формате: [Длина строки (4 байта)] + [Символы строки].
    Метод последовательно считывает все строки, перемещает их в id\_to\_string\_ и связывает со string\_to\_id\_. Значение unsaved\_start\_index\_ устанавливается равным количеству загруженных строк, чтобы предотвратить их повторную запись на диск.

2.  Поиск или добавление строки (get\_or\_add\_string)

  - Сначала выполняется поиск строки в хэш-таблице string\_to\_id\_ за время O(1). Если строка найдена, метод сразу возвращает её существующий ID.
  - Если строка новая, она добавляется в конец id\_to\_string\_.
  - В хэш-таблицу добавляется ссылка (string\_view) на новую строку и ей присваивается значение next\_id\_.
  - next\_id\_ увеличивается на 1.
  - Важно: запись на диск в этом методе не производится (операция выполняется только в оперативной памяти для максимальной скорости).

3.  Чтение строки по ID (get\_string)
    Метод принимает числовой ID и возвращает строку, обращаясь напрямую к id\_to\_string\_[id]. Проверка выхода за пределы массива гарантирует защиту от обращений к несуществующим данным. Время выполнения — O(1).

4.  Пакетное сохранение (flush)

  - Сравнивает unsaved\_start\_index\_ и next\_id\_. Если они равны, новых данных нет, и метод завершает работу.
  - Файл словаря открывается в режиме дозаписи (std::ios::app | std::ios::binary).
  - В цикле от unsaved\_start\_index\_ до next\_id\_ новые строки записываются на диск в бинарном формате.
  - По завершении записи unsaved\_start\_index\_ обновляется.
  - Метод автоматически вызывается в деструкторе StringPool, что гарантирует сохранность данных при штатном завершении работы СУБД.

Взаимодействие с другими модулями:

  - Core / Executor (в будущем) - перед тем как отправить новую запись (Record) в RecordManager, движок выполнения запросов передает текстовые поля в StringPool, получает их числовые ID, и формирует компактную структуру Record. При выполнении команды SELECT происходит обратный процесс.
  - File System - пул самостоятельно управляет своим бинарным файлом, не используя абстракцию Pager, так как данные строк не имеют страничной структуры и дописываются последовательно (append-only).

Особенности реализации, важные для защиты:

  - Zero-Copy архитектура. За счет комбинации std::deque и std::string\_view текстовые данные хранятся в оперативной памяти в единственном экземпляре. Это кардинально снижает потребление RAM при работе с большими текстовыми массивами.
  - Пакетная запись (Batching) и отложенный I/O. Исключение операций с жестким диском из метода get\_or\_add\_string устраняет главное "узкое горлышко" базы данных при массовой вставке строк (например, при транзакциях или загрузке данных).
  - Устойчивость к нулевым байтам. Формат хранения с явным указанием длины строки перед её содержимым позволяет СУБД безопасно хранить любые бинарные данные и строки, содержащие символ \0.
