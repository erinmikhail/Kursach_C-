Подсистема хранения: Менеджер страниц (Pager)

Что из себя представляет
Pager — это самый низкоуровневый компонент СУБД, абстракция над операционной системой. Он скрывает работу с сырыми байтами файла и предоставляет остальной программе интерфейс для работы с массивом страниц фиксированного размера (по умолчанию 4096 байт). 

Что хранит
На диске: Бинарный файл (data.bin или column_id.idx), состоящий из идущих подряд блоков по 4 КБ. Никаких разделителей между страницами нет.
В оперативной памяти: Объект Pager хранит только путь к файлу, открытый дескриптор файла (std::fstream) и кэшированное значение общего количества страниц (num_pages), чтобы не вычислять размер файла при каждом запросе. Сами данные страниц в памяти он не хранит (этим позже займется Buffer Pool).

 Как происходят вычисления
Так как все страницы имеют строго фиксированный размер, поиск нужной страницы имеет сложность O(1). 
Для вычисления физического адреса страницы в файле используется формула:
Смещение_в_байтах = Номер_страницы * PAGE_SIZE
(где нумерация страниц начинается с 0).

Как всё работает
1. Инициализация: При создании объекта Pager открывает файл в бинарном режиме на чтение и запись. Если файла нет — создает его. Размер файла делится на 4096 для подсчета существующих страниц.
2. Чтение (read_page): Программа запрашивает страницу (например, №3). Pager смещает указатель чтения (seekg) на байт 3 * 4096 = 12288, считывает блок из 4096 байт в std::vector<char> и возвращает его.
3. Запись (write_page): Принимает номер страницы и массив из 4096 байт, смещает указатель записи (seekp) и перезаписывает блок на диске, после чего принудительно выталкивает данные из буфера ОС (flush), гарантируя их физическое сохранение.
4. Расширение (allocate_page): Если нужно больше места, Pager увеличивает счетчик num_pages и логически создает новую страницу в конце файла.
