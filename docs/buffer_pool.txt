BufferPool

Назначение
Buffer Pool хранит ограниченное число страниц в оперативной памяти и уменьшает
число обращений к диску. Источник страниц на диске - класс Pager.

Основные структуры
- pages_ - массив фреймов в памяти. Каждый фрейм хранит данные страницы, page_id,
  pin_count и признак is_dirty.
- free_frames_ - список свободных фреймов, которые еще не заняты страницами.
- page_table_ - отображение page_id -> frame_id для быстрого поиска страницы в кеше.
- lru_list_ - список кандидатов на вытеснение. В нем находятся только страницы
  с pin_count == 0. Самая старая страница лежит в начале списка.

Логика работы
1. fetch_page(page_id)
   Если страница уже есть в памяти, BufferPool увеличивает pin_count и убирает
   фрейм из LRU, если страница была доступна для вытеснения.
   Если страницы нет, BufferPool берет свободный фрейм или вытесняет LRU-страницу.
   При вытеснении грязная страница сначала записывается на диск.

2. unpin_page(page_id, is_dirty)
   После завершения работы со страницей pin_count уменьшается.
   Если страница изменялась, выставляется is_dirty = true.
   Когда pin_count становится равен 0, страница добавляется в конец lru_list_.

3. new_page(out_page_id)
   Через Pager создается новая страница на диске, затем под нее выделяется фрейм
   в памяти. Новая страница инициализируется нулями и возвращается уже закрепленной
   в BufferPool.

4. flush_page(page_id) и flush_all_pages()
   Эти методы записывают измененные страницы из оперативной памяти на диск.
   flush_all_pages() вызывается в деструкторе, чтобы при штатном завершении
   не потерять изменения.

Что важно для защиты
- Pin/Unpin не дает вытеснить страницу, пока с ней кто-то работает.
- LRU выбирает на вытеснение наименее недавно использованную незакрепленную страницу.
- Dirty flag позволяет не писать страницу на диск без необходимости.
